# AI 前端自动化测试浏览器扩展

> 一个让 AI 同时读取项目源码并操作浏览器、自动完成前端自动化测试的 Chrome Side Panel 扩展。

## 项目简介

现有 AI 测试方案普遍存在割裂：**能读源码的不会操作浏览器，能操作浏览器的读不到源码**。
本扩展通过 Side Panel 载入用户上传的项目源码，并结合页面 DOM 快照、视觉截图、CDP 真实鼠标键盘和网络录制，
让 AI 把「源码里的意图」和「页面上的元素」对齐，自动执行点击、输入、断言等测试动作。

## 解决的痛点

- AI 测试工具（browser-use / Stagehand 等）能操作浏览器，但读不到项目源码，只能在 DOM 上「猜」。
- AI 编程助手（Cursor / Copilot 等）能读源码，但无法操作浏览器、无法验证页面真实行为。
- 两者割裂导致测试用例与实际页面行为难以对齐。

## 核心工作流

1. 前端开发者打开被测页面，并在扩展侧边栏选择目标标签页。
2. 在扩展面板中填入自己的 **AI API Key 与 URL**。
3. 上传项目源码，输入**原始需求与测试用例**。
4. AI 读取上传源码，Content Script 只负责观察页面，点击/输入/滚动等写操作全部通过 CDP 真实鼠标键盘完成。

## 技术选型速览

| 维度 | 选型 | 说明 |
|---|---|---|
| 扩展形态 | Manifest V3 + Side Panel | 侧边栏选择目标 tab 并编排测试 |
| 源码读取 | 用户上传源码目录 | 支持 `.js/.vue/.ts/.jsx/.tsx/.css/.json` |
| 浏览器操作 | `chrome.scripting` 观察 + `chrome.debugger` 执行 | DOM 快照、元素标注、截图、网络录制、真实鼠标键盘输入 |
| AI 协议 | OpenAI 兼容 tool use | 支持自定义 base_url，兼容本地/云模型 |
| 元素定位 | 视觉标注 + label/ARIA + 预设操作模板 | 下拉、多选、表单优先走确定性模板，操作后验证 |

## 实现路线

1. **Side Panel 编排**：选择目标 tab、上传源码、配置模型、展示执行日志。
2. **观察层**：Content Script 提供 DOM 快照，`chrome.scripting` 在主世界只读检查页面状态。
3. **执行层**：CDP 执行真实鼠标、键盘、滚轮、截图和网络录制，失败不做脚本动作兜底。
4. **模板层**：下拉、多选、输入、按钮、弹窗等常见动作封装为可验证的确定性操作。
5. **Agent Loop**：AI 读取源码和页面状态，调用工具执行测试，记录断言并防止死循环。
