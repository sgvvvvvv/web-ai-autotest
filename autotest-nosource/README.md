# AI 黑盒 Web 测试扩展

这是一个 Chrome Side Panel 扩展，用于在不知道源码的情况下测试已部署的 Web 站点。它只使用生产页面中可见的运行时信息：DOM/无障碍语义、页面文本、导航、截图和网络响应；所有点击、输入、滚动和拖拽均通过 Chrome Debugger Protocol 发送真实鼠标键盘事件。

## 使用方式

1. 在 Chrome 的 `chrome://extensions` 启用开发者模式，并加载本目录。
2. 打开目标生产站点，点击扩展图标打开侧边栏。
3. 配置 OpenAI 兼容的 API URL、Key、模型；模型支持图片时勾选视觉能力。
4. 选择目标标签页，点击“勘探当前站点”。扩展会读取当前页面的导航和可交互控件，不读取本地文件或站点源码。
5. 填写业务需求后点击“生成黑盒用例”，或直接粘贴 CSV 用例。
6. 点击“运行真实交互测试”。测试过程记录断言、DOM 状态、截图和网络响应。

视觉截图会从当前用例的标题、步骤和预期中提取关键词，只标注匹配的控件及其所在区域；命中区域集中时自动裁剪，无法关联时才回退到少量全局候选。UI 验证截图使用同一焦点区域但不绘制标注。

测试环境默认允许真实执行创建、编辑、删除、提交和数据清理等 CRUD 流程。支付、外部发送和第三方不可逆操作仍只验证入口。运行时覆盖区会持续显示已发现页面、控件、实际交互数及用例完成度。

## 覆盖边界

黑盒测试覆盖从当前页面可访问到的导航路径、已渲染控件及其可见交互。它无法证明未暴露在 UI 中的隐藏功能，也不会猜测账号权限、未公开路由或后端实现。对于删除、支付、提交等不可逆操作，生成的用例只验证入口、提示和二次确认；需要执行实际业务写入时应由测试账号和明确的测试数据策略控制。

## 结构

- `content/content.js`：只读采集运行时 DOM、导航和页面文本。
- `core/visual-controller.js`：通过 CDP 执行真实输入和截图。
- `core/network-recorder.js`：记录页面网络请求/响应。
- `core/agent-loop.js`：黑盒测试 Agent 循环、断言和停滞保护。
- `core/coverage-tracker.js`：运行时页面、控件、真实交互与用例覆盖统计。
- `sidepanel/`：站点选择、运行时勘探、用例生成和测试执行 UI。

源码上传、源码检索、源码分析和项目架构分析模块已移除。

## 验证

运行 `npm test` 和 `npm run check`。项目不需要安装依赖。
